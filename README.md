# Канал «Физика за 5 минут» — пайплайн на Python

Короткие (3–5 мин) ролики по физике для 7–8 класса. Озвучка — Silero
(офлайн, GPU), анимация — Manim, всё локально. Цель — вовлечение аудитории,
без продажи курсов.

> Всё (Python, venv, кэши, модели, рендер) лежит на **D:**, потому что на
> C: свободно ~2 ГБ. Окружение задаётся файлом `d:\ggpolole\env.ps1`.

## Установка (PowerShell)

```powershell
. d:\ggpolole\env.ps1            # кэши -> D:
$py = 'd:\ggpolole\.venv\Scripts\python.exe'

# torch с CUDA 12.1 (драйвер 551.86, GTX 1050 Ti = Pascal sm_61):
& "$env:UV" pip install --python $py --index-strategy unsafe-best-match `
    --extra-index-url https://download.pytorch.org/whl/cu121 "torch==2.5.1+cu121"

# остальное:
& "$env:UV" pip install --python $py manim==0.18.1 imageio-ffmpeg==0.5.1 `
    soundfile==0.12.1 PyYAML==6.0.2 "numpy<2.0"
```

## Сборка ролика

```powershell
. d:\ggpolole\env.ps1
cd d:\ggpolole\physics_channel
& d:\ggpolole\.venv\Scripts\python.exe -m pipeline.build_video 001
```

На выходе в `out\001\`:
`001_*.mp4` (видео + озвучка), `001.srt` (субтитры),
`001_thumbnail.png` (обложка), `metadata.json` (заголовки/описание/теги).

## Как это работает

1. `core/tts_silero.py` — каждая фраза → wav, замер длительности.
2. `out/<ep>/timing.json` — длительности; `core/scene_base.py` подгоняет
   каждый «бит» Manim ровно под длину его аудио → голос и картинка синхронны.
3. `pipeline/build_video.py` — склейка озвучки, рендер Manim, ffmpeg-микс,
   `.srt`, обложка, `metadata.json`.
4. `content/topics.yaml` — план на 100 роликов (8 серий-плейлистов).
5. ffmpeg берётся из пакета `imageio-ffmpeg` (копия в `.bin\ffmpeg.exe`,
   систему/C: не трогаем).

## Достоверность фактов

Перед продакшеном каждый числовой/исторический факт сверяется по ≥2
независимым авторитетным источникам и фиксируется в
`content/scripts/<ep>_*.md` (таблица факт-чека). Для №1 источники —
NASA, NIST, Wikipedia, Physics World.

## Плейбук под алгоритмы YouTube (как загружать ради просмотров)

**Главное: CTR (клик) и удержание решают рост. Ниша + время сессии — то,
что любит алгоритм.**

- **Серии-плейлисты.** Каждое видео — в своей серии (`series` в
  topics.yaml → плейлист в `pipeline/seo.py`). Плейлисты и конечные
  заставки «следующий выпуск» максимизируют время сессии.
- **Обложка.** 1–3 крупных слова, высокий контраст, единый стиль,
  читается в мини-размере (генерится автоматически).
- **Заголовок.** Любопытство + конкретика + ключевик в начале, ≤ ~60
  символов. В `metadata.json` 2–3 варианта — тестировать по CTR.
- **Хук 0–5 сек.** Сразу вопрос + обещание ответа, без длинных интро.
- **Хронометраж 3–5 мин.** Плотно, без воды → высокий % досмотра.
- **Расписание.** Стабильно 2–3 ролика/неделю (батч + отложенная
  публикация). Для РФ-аудитории — будни вечер / выходные день.
- **Метаданные.** Первые 2 строки описания — ключевые; загрузить `.srt`
  как субтитры ru (SEO + досмотр без звука); 5–8 тегов; закреплённый
  комментарий-вопрос; карточка в середине → связанный ролик.
- **Первые 24–48 ч.** Быстро отвечать на комментарии — важный сигнал.
- **Shorts-воронка (опция).** Вертикальный тизер 30–50 сек из тех же
  ассетов → рост подписчиков для нового канала.
- **Не делать.** Кликбейт во вред досмотру, копирайт-музыка, длинные
  интро, «прыжки» по нишам.

## Загрузка (ручная или авто)

`metadata.json` содержит готовые поля и `publish_notes` — чек-лист
публикации. Автозагрузку через YouTube Data API v3 (OAuth) можно добавить
модулем `upload/youtube_upload.py` (опционально, требует Google-аккаунт).
